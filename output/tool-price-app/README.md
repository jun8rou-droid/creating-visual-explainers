# 🔧 工具費くらべ（MVP）

町工場向けの「工具価格管理アプリ」です。
**写真を撮る → 品番を確認 → 価格を入れる → 保存**、これだけで登録が完了します。
同じ品番の過去価格や、カテゴリごとの価格比較を見られます。

サーバー不要・インストール不要。**HTMLファイル1つだけ**で動きます。

---

## 起動方法

### いちばん簡単（PCで試す）
1. `index.html` をダブルクリックしてブラウザで開く
2. 最初の画面で「**サンプルデータを入れて試す**」を押すと、動作を確認できます

> ⚠️ カメラ撮影やスマホで使う場合は、下記の「ローカルサーバー経由」をおすすめします（一部ブラウザは`file://`でカメラが使えません）。

### ローカルサーバー経由（スマホでも使う）
ターミナルで `index.html` のあるフォルダに移動して、どれか1つ実行：

```bash
# Node.js がある場合
npx serve

# Python がある場合
python3 -m http.server 8000
```

表示されたURL（例 `http://localhost:8000`）を開きます。
**同じWi‑Fi内のスマホ**からは、PCのIPアドレス（例 `http://192.168.0.10:8000`）で開けます。

## 使い方

1. ホームの大きな「📷 写真で登録」ボタン、または右下のカメラボタンを押す
2. 工具ラベル・箱・納品書などの**写真を撮る/選ぶ**
3. 自動で**品番候補**が出るので、正しいものをタップ（出ない・違う時は手入力）
4. **価格**を入力して「💾 保存」
   - 工具屋・メーカー・カテゴリ・購入日・メモは「任意項目」を開けば入力可（省略OK）
   - 購入日を空にすると「今日」が自動で入ります
5. 保存すると**品番詳細**に移動。最新/最安/最高/前回比、工具屋別比較、価格履歴が見られます

## 機能

| 画面 | 内容 |
|------|------|
| ホーム | 写真で登録ボタン・最近登録した工具・値上がりしている工具・カテゴリ別 |
| 登録 | 写真→品番候補→確定→価格→保存（必須は品番・価格・写真のみ） |
| 品番詳細 | 最新/最安/最高価格・前回比・工具屋別比較・価格履歴・登録写真一覧 |
| カテゴリ | 一覧／追加。カテゴリ内の件数・合計・平均・よく買う品番・値上がり品番 |
| 検索 | 品番・メーカー・工具屋のキーワード検索＋カテゴリ絞り込み |

### 品番読み取り（OCR）
- [Tesseract.js](https://tesseract.projectnaver.com/) を使い、**端末内で**写真から品番候補を抽出します（画像は外部に送信しません）
- 初回はOCR辞書の読み込みで数秒かかります
- ネットが無い・読み取れない時は手入力にフォールバックします

### カテゴリ（初期値）
ドリル / チップ / タップ / エンドミル / ホルダー / 測定具 / その他
→ カテゴリ画面からあとで追加できます。

## データの保存先

すべて**お使いのブラウザ内（IndexedDB）**に保存されます。画像も含めて端末内のみ。
サーバーには何も送りません。

> 注意：別の端末・別のブラウザとはデータを共有しません。ブラウザの「サイトデータ削除」をすると消えます。

### データベース設計（IndexedDB: `toolPriceDB`）

| ストア | キー | 主な項目 |
|--------|------|----------|
| `tools` | partNumber | 品番・メーカー・カテゴリ・メモ・登録日 |
| `prices` | id(自動) | 品番・**価格**・工具屋・購入日・画像ID・カテゴリ・登録日時 |
| `images` | id(自動) | 画像Blob（長辺1280pxにリサイズ） |
| `meta` | name | カテゴリ一覧・工具屋/メーカーの入力候補 |

設計の考え方：品番を集約キーにし、価格は時系列で `prices` に積みます。
最新・最安・最高・前回比は、そのつど `prices` から計算します（集計を持たず常に正しい値）。

## 技術構成
- 単一HTML（HTML/CSS/JS内包）
- データ：IndexedDB（端末内）
- OCR：Tesseract.js（CDN・端末内処理）
- スマホ前提のレスポンシブUI（PCでは2カラム表示）
