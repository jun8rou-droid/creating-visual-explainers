# 週次売上レポート自動化 — 用語集（新卒向け）

このプロジェクトの説明・README・チェックリストに出てくる言葉を、**初めて聞く人向け**に並べています。横に短いたとえを添えています。

---

## データとファイル

### CSV（シーエスブイ）

**意味:** 表のデータを、カンマ（`,`）で区切って並べたテキストファイル。Excel で「CSV 形式で保存」するとよく出てきます。

**なぜ使うか:** 受注管理ツールから「この週の売上一覧」をファイルで受け取りやすいからです。

**覚え方:** 「Comma-Separated Values＝カンマ区切りの値」。

---

### 文字コード（UTF-8 / CP932 など）

**意味:** 日本語や記号を、コンピュータが読むための「文字の地図」の違いです。同じ「あ」でも、地図が違うと文字化けします。

**UTF-8:** いまよく使われる世界標準に近い形式。英語のドキュメントや Web と相性がよいです。

**CP932（シフト JIS 系）:** 日本の Windows で昔からよく使われてきた形式。**国内の業務ソフトから出る CSV は CP932 であることが多い**です。

**このプロジェクトでは:** 日勝ネジ様のサンプル CSV は CP932 で読む設定にしています。

---

### SQLite（エスキューライト）

**意味:** 1 つのファイルにまとめて保存できる、小さなデータベース（DB）のソフトの名前です。サーバを別途立てなくても使えます。

**なぜ使うか:** 取り込んだ行・過去のデータをここに蓄え、**「同じ顧客・同じ品目の過去の単価」**を検索するためです。

**覚え方:** 「Lite＝軽量」。本番の大規模システムの代わりではなく、**このレポート用ツールの控え**という位置づけです。

---

### 正本（ソース・オブ・トゥルース）

**意味:** 「この数字が本当か」と言われたときに、**いちばん正しいとみなすデータの置き場**（いつもの受注管理ツール本体）です。

**このツールの SQLite:** 正本の**コピー**を加工・集計するためのもので、**会社の公式帳簿の代わりにはなりません**。

---

## プログラムと実行

### コマンドライン（ターミナル）

**意味:** 黒い画面（ターミナル）に `python ...` のように文字を打って、プログラムを動かすやり方です。

**なぜ最初はここか:** 画面付きアプリを作る前に、**まず処理が正しいか**を確かめやすいからです。

---

### `--dry-run`（ドライラン）

**意味:** 「本番と同じ準備はするが、**一番お金がかかる／外部につながる処理だけ飛ばす**」実行モードです。

**このツールでは:** Claude（Anthropic）の API を**呼ばず**、送る予定の内容（JSON）だけ表示します。動作確認や誤送信防止に使います。

**覚え方:** Dry＝水を通さない＝本番の「水」＝API 呼び出しを通さない。

---

### `--week`（週キー）

**意味:** 「この取り込みは**どの週の分か**」を表すラベルです。例: `2024-W01`（2024 年の第 1 週）のように書きます。

**なぜ必要か:** 同じ週を二重に取り込まない、**再実行したときに上書きする**などの整理に使います。

---

### `run_id`

**意味:** そのときの実行を区別するための **1 回ごとの ID**（UUID という長いランダム文字列）です。

**なぜ必要か:** ログやトラブル調査で「この実行の結果はどれか」を追いやすくするためです。

---

### `--config`（設定ファイル）

**意味:** 「CSV の列名がツールの想定と違うとき」に、**どの列が顧客コードで、どの列が金額か**を書いた JSON ファイルのパスです。

**例:** `config/nichisho_nissyo.json` は、日本語ヘッダの CSV を内部形式に変換するルールが書いてあります。

---

## AI と API

### API（エーピーアイ）

**意味:** プログラムから「こちらから依頼すると、向こうが結果を返す」**窓口**のことです。Web サイトを人がブラウザで見るのではなく、**機械同士のやり取り**です。

**例:** Anthropic の API に「この文章を書いて」と送ると、AI が答えを返します。

---

### LLM（エルエルエム）／大規模言語モデル

**意味:** 大量の文章から学習した AI で、**質問に答えたり、文章を生成したり**するものです。この文脈では **Claude** がそれに当たります。

**このツールでの役割:** 売上の**足し算そのものはプログラム**がやり、LLM には **「読みやすい週次の文章」にする**ところだけお願いします。数字を捏造しにくくするためです。

---

### トークン

**意味:** AI が文章を処理するときの、**細かいかたまり**の単位です。日本語だと「1 文字＝1 トークン」ではないことが多いです。

**料金:** API の課金はだいたい「送ったトークン」「返ってきたトークン」の量に比例します。**長い CSV を丸ごと送ると高くなりやすい**ので、このツールでは短い JSON だけ送る設計にしています。

---

### プロンプト

**意味:** AI に渡す「指示文＋材料」です。こちらの書き方で、**丁寧な答え**になったり、**数字を変えない**ようになったりします。

---

### JSON（ジェイソン）

**意味:** `{ "key": value }` のように、**機械同士がデータを渡しやすい**書き方です。

**このツールでは:** 集計結果や単価候補を JSON にまとめてから LLM に渡します。

---

## 運用・仕組みの言葉

### 冪等（べきとう）

**意味:** 「同じ操作を何度しても、**壊れたり二重に積み上がったりしない**」ことです。

**例:** 同じ週の CSV をもう一度流しても、データがおかしくならないようにする、という話です。

---

### 監査ログ（けんさログ）

**意味:** 「いつ、誰が、何をしたか」を後から追える**記録**です。

**なぜ大事か:** トラブルや確認依頼が来たときに説明しやすいからです（まだ MVP では未実装の部分もあります）。

---

### LINE Messaging API

**意味:** LINE の公式が提供する、**プログラムから LINE にメッセージを送る**ための窓口（API）です。

**このプロジェクトでは:** いまは「文案まで」が中心で、**実際に LINE で送る**ところは次の拡張として想定されています。

---

## すぐ上の階の言葉（聞くことがあるかも）

### cron（クロン）

**意味:** Linux などで「**決まった時間にプログラムを動かす**」仕組みの定番です。

**例:** 毎週月曜 9:00 に週次スクリプトを実行する、など。

---

### GitHub Actions

**意味:** GitHub が提供する「**クラウド上で決まった手順を実行**」する機能です。cron のように定期実行もできます。

## 手動の CLI と本番の「届き口」（新卒向け）

### いまの CLI（シーエルアイ）

**意味:** エンジニアがターミナル（黒い画面）に **`python ... --csv ファイル名 --week ...`** と**自分で打って**、ツールを一度動かすやり方です。試す・確認するには向いています。

### 本番の「届き口」

**意味:** 本番では、**人が打たなくてよい**ようにします。毎週決まった時間に CSV が**共有フォルダやクラウドの受け皿**などに届き、ツールが**「届いた」ことに気づいたら**、いまと同じ流れ（読む・集計する・文章を頼む）を**自動で**始めます。  
**レシピ（処理の中身）は同じ**で、**「スタートの合囲」がキーボードから「ファイルが届いた」に変わる**と考えると分かりやすいです。

### `--week` を自動にする話

**意味:** いまは人が「第何週の分です」と `--week` に書いています。本番では、**ファイルの日付や運用ルールから、プログラムが週の名前を決めて**同じ欄に入れる、という**別の小さな作業**が残ります（すぐにやらなくてもよい、と割り切ってよいことが多いです）。
