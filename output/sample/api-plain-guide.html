APIって何？

売上レポートやLINE通知の話をするときによく出る「API」を、専門用語なしで整理した図解です。

1 ひとことでいうと

APIは、プログラム同士が、決まった手順でお願いしあう「窓口」のことです。

人がブラウザでボタンを押す代わりに、自動処理のプログラムがその窓口に「こうして」と短いメッセージを送り、向こうのコンピュータが答えを返します。

3 あなたが作りたい仕組みでの「API」

週に1回CSVを取り込んで、気づきを文章にしてLINEで知らせる、という流れだと、APIが出てくるのは主に2か所です（※CSVファイルを手で置くだけなら、その部分はAPIじゃなくてもよい）。

流れと窓口
① 受注管理ツール → あなたのプログラム よくあるのはファイル

CSVをエクスポートしてフォルダに置くだけなら、これは「API」ではなくファイルの受け渡しです。ツールが自動で送ってくれる専用の窓口が付いている場合だけ、その窓口がAPIと呼ばれます。

② あなたのプログラム → AI（文章を作ってもらう） API がよく使われる

「今週の数字の要約を、秘書みたいに書いて」とプログラムからお願いするとき、そのお願い口がAIのAPIです。ブラウザでChatGPTを開いて打つ操作を、機械が代わりに行うイメージです。

③ あなたのプログラム → LINE（通知を送る） API がよく使われる

自分のスマホのLINEに、プログラムからメッセージを送るには、LINE側が用意した決まった送り方（Messaging API）を使います。これもAPIの仲間です。

4 「APIじゃない」と言われることもある

  • ・ 人がブラウザで画面をクリックするだけ（プログラムが介在しない）は、普通APIとは言いません。
  • ・ CSVファイルを手で保存して、フォルダに置いただけなら、それは「ファイル」であって、必ずしもAPIではありません。

まとめ

  • API＝プログラム向けのお願いの窓口（決まった言い方で頼む）
  • 売上レポート自動化では、AIに文章を頼む窓口と、LINEに送る窓口が「API」としてよく出てくる
  • CSVはファイルの受け渡しであり、APIと混同しないと整理しやすい
